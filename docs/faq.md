# FAQ（用户/玩家）

!!! tip ""
    遇到问题时，优先核对“昵称 + 邀请码 + 学生凭证 + 网络状态”，大多数登录与断线问题可快速定位。

---

## 账号与登录

- **如何开始游戏？**
  进入首页，填写昵称和邀请码。新玩家通过后会进入角色创建页，选择专业并分配初始属性。
- **老玩家如何登录？**
  使用原昵称 + 邀请码 + 学生凭证登录。登录后可以选择加载已有存档，或开始新游戏。
- **学生凭证是什么？**
  新玩家首次登录时系统生成的长期凭证，用于证明你是该昵称的持有人。请妥善保存。
- **登录凭证遗失怎么办？**
  需要联系管理员重置，或使用新的昵称重新开始。
- **提示黑名单或受限怎么办？**
  账号或凭证被列入黑名单/限制，请联系管理员申诉。

## 角色与专业

- **专业什么时候选择？**
  新玩家登录后在角色创建页手动选择专业，不再由系统随机或考试分配。
- **初始属性如何分配？**
  `IQ`、`EQ`、`Luck` 三项基础值总和必须为 `250`，每项范围为 `50-150`。
- **专业加成怎么算？**
  专业的 IQ 增益会在你分配的基础 IQ 之上额外叠加。
- **开始新游戏会影响旧存档吗？**
  开始新游戏会重新初始化当前游戏状态。若想保留进度，请先在游戏内保存旧存档。

## 存档与进度

- **存档在哪里？**
  运行时状态在 Redis，持久化存档在 PostgreSQL。手动保存、保存退出和学期切换会写入持久化存档。
- **换设备会丢进度吗？**
  使用同一昵称、邀请码和学生凭证登录后，可以从存档选择页加载已保存进度。
- **掉线会丢数据吗？**
  可能丢失最近一次保存后的少量进度，建议在关键节点手动保存。

## 大模型（LLM）

- **能否使用自定义模型？**
  可以。在登录页展开“自定义大模型”，填写模型代号和 API Key；仅本会话使用，不落库。
- **API Key 安全性？**
  仅保存在浏览器会话内并随 WebSocket 首条消息传递，服务器不持久化；请务必在 HTTPS 环境下使用。
- **未填写会怎样？**
  将使用平台默认模型。若 LLM 不可用，AI 模式会自动回落到混合模式。

!!! tip ""
    配置第三方 API Key 前，建议先阅读[模型配置](user/models.md)中的安全须知。

## 钉钉私聊

- **为什么钉钉联系人列表是空的？**
  只有已经发过消息的角色才会出现。还没有触发钉钉消息时，联系人列表为空是正常的。
- **红点是什么意思？**
  表示该联系人有未读新消息。点开联系人后会标记为已读。
- **为什么有些角色不能回复？**
  只有室友、同学、朋友、助教、老师、暗恋对象等角色类型支持回复机制；其他角色可能只发送通知或剧情消息。
- **回复会影响属性吗？**
  同一名角色每 3 次玩家回复算一轮对话，NPC 第三次回复后可能轻量影响精力、心态、压力、EQ、运气或声望。
- **学期切换后聊天记录会消失吗？**
  不会。钉钉联系人和私聊记录会随游戏状态保留；为了性能，系统可能只保留最近若干条消息。

详见[钉钉私聊](user/dingtalk.md)。

## WebSocket 与心跳

- **为什么断线/被踢？**
  同一账号仅允许单连接；30s 心跳超时会清理；消息频率需 ≥50ms 间隔。
- **看到“auth_error”提示？**
  首条 WebSocket 消息里的 JWT 无效、账号受限，或选择的存档不存在。
