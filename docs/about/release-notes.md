# 发行说明

本页记录玩家和部署者需要知道的主要变化。更细的开发决策见开发人员指南中的接手报告与开发计划。

## 2026-06-30

### 序章节奏优化

- 登录前序章保留前两句献词，并改为两句依次闪现。
- 主体故事改为日记本逐行书写效果，中段翻页后进入十八岁返校的收束场景。
- 序章仍然可跳过，并继续在播放期间阻止登录/存档分流和游戏 WebSocket 提前启动。

## 2026-06-20

### 属性与道具扩展生产线

- 新增 `world/stat_definitions.json` 作为属性定义事实源，统一初始分配、道具 effect、事件/钉钉 effect 和前端展示元数据。
- 角色创建接口新增兼容式 `stats` 映射，旧的 `iq` / `eq` / `luck` / `charm` 字段继续可用。
- HUD、右侧状态卡、道具页、结局页和新手引导已经接入生成的属性元数据，不再手写属性标签、默认值和上限。
- 后端初始值、Redis 安全更新、道具 effective stats、事件库筛选和内容生成上下文统一读取属性定义中的默认值与范围。
- 新增世界数据校验与属性脚手架脚本，后续新增普通道具或属性时可减少重复手工修改。
- 管理后台新增“道具配置”页面，可图形化调整 `world/items.json` 的金币经济、道具价格、标签和被动加成，保存后热重载并记录审计。

## 2026-06-19

### 魅力属性

- 角色创建新增 `魅力`（`charm`）初始属性，与 IQ、EQ、Luck 一起分配。
- 初始属性预算从 250 调整为 300；四项属性每项仍为 50-150，专业 IQ 加成仍在预算校验后额外叠加。
- 魅力已纳入游戏常规状态：HUD、右侧面板、结局页、事件/钉钉结算、道具加成和内容生成上下文都会识别该属性。

### 结算与反馈修复

- GPA 显示统一为累计学分加权 GPA；期末成绩单继续单独展示本学期 GPA。
- 钉钉联系人列表增加上限与旧联系人复用，避免每轮对话都新增 NPC。
- 休闲动作结果弹窗会列出实际数值变化；正向收益到达端点时会尽量转移到其他可提升属性，健身可小概率提升魅力。
- 成就解锁会即时提示，期末成绩单展示本学期新解锁成就，毕业典礼展示具体成就详情。
- Game Over 与毕业页新增“回到首页”入口，方便退出当前局重新登录或选档。
- 重新设计 Game Over 与毕业典礼视觉：退学页改为暗色日记本氛围，毕业页使用校园风景轮播，并按最终 GPA 显示不同的重生呼应文案。

## 2026-06-18

### 道具与金币

- 主界面中央面板新增“道具”标签页，可搜索、购买和出售道具。
- 新增金币状态；新游戏有初始金币，期末结算会按 GPA 与挂科数发放金币，随机事件和钉钉结算也可能影响金币。
- 道具持有即生效，每种道具同一时间最多拥有一件；出售会回收金币并移除对应加成。

### 数值平衡与模型成本控制

- 新学期开始时会把精力向满值回调一半，避免低精力跨学期后直接进入不可恢复开局。
- 登录页新增钉钉 RP 专用 MiniMax API Key 配置；填写后钉钉私聊使用玩家自己的 M2-her key。
- 玩家配置通用自定义 LLM 但未配置 RP Key 时，钉钉不再继续消耗平台默认 M2-her，而是回退到通用自定义 LLM。

## 2026-06-09

### 文档站重构

- 文档站从 MkDocs Material 迁移到 VitePress，文档主题、组件和静态资源全部收束在 `docs/` 内维护。
- 首页重做为星空氛围页，加入打字机标题和可交互的游戏预览 demo。
- 独立游戏预览页移除，交互式 mock demo 收束到首页；demo 只使用模拟状态，不连接 WebSocket 或后端 API。
- 文档配色切换为深蓝黑暗色模式与亮蓝白亮色模式，不再沿用 Material 默认蓝色风格。

### 生产运行与资源占用

- 生产环境后端默认关闭 SQL echo，并跳过启动时 `Base.metadata.create_all`；数据库结构由 Alembic `migrate` 服务负责。
- 后端运行路径增加资源节省优化，包括 Redis 连接池复用、静态世界数据缓存、MiniMax HTTP client 复用，以及更适合 2C2G 服务器的启动行为。
- 生产 `docker-compose.yml` 不再把后端 `8000` 端口发布到公网宿主机，Nginx 通过 Docker 内网访问 `backend:8000`；本地开发仍由 `docker-compose.override.yml` 提供 `127.0.0.1:8000:8000` 映射。
- 文档站移除旧的手工资源下载包入口；世界观和数值资源以仓库中的 `zjus-backend/world/` 最新文件为准。

## 2026-06-07

### 登录前序章

- 首次访问游戏时会先播放一段可跳过的序章，再进入登录、存档选择或角色创建流程。
- 序章状态保存在浏览器 `localStorage.zjus_prologue_seen_v1` 中；看过或跳过后，刷新与返校登录不会重复播放。
- 序章播放期间不会提前进入登录/存档分流，也不会建立游戏 WebSocket。
- 序章画面加入多张校园背景图，并在“明天会更好”相关台词处切换到夕阳图。

## 2026-06-06

### 钉钉私聊升级

- 钉钉从简易公告式消息升级为联系人私聊：每个角色拥有独立会话，未发过消息的角色不会出现在联系人列表。
- 有新消息的联系人会显示红点；点开联系人后标记为已读。
- 联系人和聊天记录会随游戏状态保留，学期切换不会清空。
- 室友、同学、朋友、助教、老师和暗恋对象支持玩家回复；每条 NPC 新消息可生成若干回复选项。
- 同一角色每三次玩家回复算一轮对话，NPC 第三次回复后由 AI 结算轻量属性影响。

### 流程与界面修复

- 修复进入下学期后前端仍停留在旧学期、需要手动刷新才能看到新状态的问题。
- 调整右侧游戏速度/控制区布局，避免钉钉会话拉长后遮挡消息内容。
- 修正同学、室友等可回复联系人误显示“暂不支持回复”的提示。
- 清理后端 Pylance 与 ruff 诊断中的类型噪声，保持运行逻辑不变。

## 历史更新

### 内容与检索架构升级

- 内容系统从运行时大模型直生切换为“离线预构建 + 运行时检索”。
- 新增预构建产物：`world/event_library.json`、`world/cc98_library.json`、`world/query_embeddings.json`、`world/character_embeddings.csv`。
- 事件和 CC98 内容优先使用本地库，LLM 作为兜底。
- 钉钉角色内容优先使用 pgvector 相似度检索，检索或模型不可用时降级到随机角色或通用 LLM。
- Docker 启动链路加入 `seed_embeddings` 一次性服务，在 `migrate` 后导入角色向量。

### 初始版本

- 建立 Vue 3 + FastAPI 的校园模拟器主循环、课程、属性、随机事件、存档和文档站基础结构。
