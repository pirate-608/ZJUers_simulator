# Shared `.codex/` in `ZJUers Simulator` Project.

This directory contains Codex-specific configuration for this collaborative project.

## Contents

- `skills/` – Custom skills (markdown files) that extend Codex's capabilities for project-specific tasks (e.g., build, test, deploy workflows).
- `rules/` – Project-specific rules or guidelines that Codex should follow.
- `openai.yml` – Codex‑specific configuration file. Other agent tools (including Claude Code) ignore this file.
- Other configuration files as generated by Codex.

## How to use

- Skills are automatically discovered when placed in `skills/`. Refer to Codex documentation for skill format.
- To share a new skill: add a `.md` file to `skills/` and commit it.
- To modify behavior for everyone: update rules or skills and push to the repository.

## For contributors

- Do not put sensitive data (keys, secrets) here – this directory is versioned.
- If you're unsure what a skill does, read its markdown file; it should contain a clear description.
- Keep skill names short and action-oriented (e.g., `run-tests.md`, `format-code.md`).
