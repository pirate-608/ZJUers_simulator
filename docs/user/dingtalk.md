# 钉钉私聊

钉钉面板用于展示游戏中角色发来的消息。它不是固定公告栏，而是会根据你的当前状态、课程进度、心态、压力、GPA 和随机上下文生成对话内容。

## 联系人列表

- 只有已经发过消息的角色会出现在联系人列表中。
- 有新消息的联系人会显示红点；点开对应联系人后会标记为已读。
- 每个角色有独立的私聊记录，不同角色的消息不会混在同一条会话里。
- 联系人与聊天记录会随游戏状态保留，学期切换不会清空。
- 为了避免记录无限增长，系统默认最多保留 12 位联系人、每位联系人最近 50 条消息。
- 新消息会优先复用已经结束一轮对话的旧联系人；达到联系人上限后，系统会强制从无打开轮次的旧联系人中选择一位继续聊天。

## 回复消息

部分角色支持玩家回复。收到这类角色的新消息后，私聊底部会出现 2-3 个可选回复，点击其中一个即可发送。

当前可回复角色类型包括：

- 室友
- 同学
- 朋友
- 助教
- 老师
- 暗恋对象

其他角色仍可能发来消息，但不会提供玩家回复选项。

## 一轮对话

玩家每回复同一名角色 3 次，视为完成一轮对话。第三次回复后，NPC 会再回复一条消息，然后系统根据这一轮互动轻量结算属性影响。

可能受到影响的属性包括：

- 精力
- 心态
- 压力
- EQ
- 运气
- 魅力
- 声望
- 金币

钉钉私聊的影响通常比较克制，可能影响精力、心态、压力、社交类属性或金币，但不会替代课程、考试、休闲动作和随机事件的主要结算。

## 内容来源

钉钉消息默认优先使用角色扮演模型生成，并参考角色设定。若你在登录页填写了通用自定义 LLM，但没有填写自定义 RP MiniMax Key，钉钉会回退到你的通用模型，避免继续消耗平台默认 RP 模型额度。若模型不可用，游戏会尽量使用降级内容维持流程；在纯算法模式下，由于当前没有钉钉本地消息库，新的钉钉消息可能会减少或跳过。

详见[内容生成模式](./content_modes.md)和[模型配置](./models.md)。
