# 内容生成模式

游戏右侧面板可以切换内容生成模式：`算法`、`混合`、`AI`。它们只影响随机事件、CC98 帖子和钉钉消息等动态内容，不会改变课程、数值结算或存档规则。

## 三种模式

| 模式 | 适合场景 | 内容来源 |
|:---|:---|:---|
| Library / 算法 | 想要稳定、快速、少消耗 API 额度 | 优先使用本地预构建内容库，不主动请求 LLM。随机事件和 CC98 帖子来自内容库；当前没有钉钉本地库，因此该模式会跳过钉钉消息。 |
| Hybrid / 混合 | 日常游玩推荐，默认模式 | 先使用本地内容库；当库未命中或内容不足，并且 LLM 可用时，再用 AI 补充生成。 |
| AI | 想要更多实时生成内容，且已经配置可用 LLM | 跳过本地事件/帖子库，直接请求 LLM 生成随机事件和 CC98 帖子；钉钉消息会优先使用角色扮演模型，失败时再走通用 LLM。 |

## LLM 不可用时

当你切换内容生成模式时，游戏会在后台检查 LLM 是否可用。如果检测到 LLM API 不可用，并且你正在使用 `AI` 模式，游戏会自动切回 `Hybrid / 混合` 模式，并显示提示。

这样做是为了让游戏继续运行：`Hybrid / 混合` 仍然可以使用本地内容库生成随机事件和 CC98 内容，只是在需要 AI 补充时会减少或跳过相关内容。已经进行的课程、考试、属性变化和存档不会因此丢失。

如果 LLM 已被判定不可用，再次点击 `AI` 模式时，游戏会保持当前模式并提示 `LLM API 不可用`。你可以继续用 `Hybrid / 混合` 或 `Library / 算法` 游玩，也可以到[模型配置](./models.md)检查 API Key、模型名称和 Base URL。
